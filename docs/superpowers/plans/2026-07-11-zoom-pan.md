# Zoom / Pan Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Zoom (1×–8×) and pan the live stack image in both the embedded Live view and the detached OBS-captured broadcast window, non-destructively, with a pan-clamp that keeps the image covering the view. Zooming the broadcast window frames the OBS stream.

**Architecture:** A pure, unit-tested `ZoomPanState` (scale + offset + clamp math) lives in `LiveAstroCore`. `AppModel` holds one shared instance; `BroadcastView` applies `.scaleEffect`/`.offset`, drives it from scroll/pinch/drag gestures + a slider + Fit/100% buttons, and auto-hides the control chrome on the detached window so the OBS capture stays clean.

**Tech Stack:** Swift 5.10, SwiftUI, XCTest. `LiveAstroCore` (Foundation/CoreGraphics) + `LiveAstroStudio` (`@Observable`).

## Global Constraints

- Swift 5.10, macOS 14+. `LiveAstroCore` Foundation/CoreGraphics only (CGSize/CGFloat are CoreGraphics — allowed).
- Core tests: `swift test --filter LiveAstroCoreTests`; existing tests stay green.
- TDD for T1 (ZoomPanState). T2 (AppModel + BroadcastView SwiftUI) is MANUAL/BUILD-VERIFIED — gestures/window lifecycle out of unit scope, per prior pillars.
- Non-destructive: zoom/pan is a view transform only; the stack, `master.fit`, snapshots, and `replay.mp4` stay full-frame (do NOT touch those paths).
- Zoom state is ephemeral: reset to `.fit` on session start; NOT persisted.
- Pan clamp: the scaled content always covers the view (no empty margins); at scale 1 no pan.
- `configuresWindow == true` = the detached OBS window (auto-hide controls); `false` = embedded Live tab (controls always visible).
- Branch `feature/zoom-pan` (off main). Never commit to main.

```swift
        let view = CGSize(width: 100, height: 100)

        let view = CGSize(width: 100, height: 100)
        let content = CGSize(width: 100, height: 100)
        XCTAssertEqual(ZoomPanState.clampedOffset(CGSize(width: 80, height: 80), scale: 2,
                       viewSize: view, fittedContentSize: content), CGSize(width: 50, height: 50))
        XCTAssertEqual(ZoomPanState.clampedOffset(CGSize(width: 30, height: -30), scale: 2,
                       viewSize: view, fittedContentSize: content), CGSize(width: 30, height: -30))
    }

    func testPerAxisClamp() {
        // landscape view, portrait content fitted to height → at scale 2 only the
        // vertical axis overflows; horizontal pan is pinned to 0.
        let view = CGSize(width: 200, height: 100)
        let content = CGSize(width: 50, height: 100)          // fitted at scale 1
        let out = ZoomPanState.clampedOffset(CGSize(width: 40, height: 40), scale: 2,
                       viewSize: view, fittedContentSize: content)
        XCTAssertEqual(out.width, 0)                          // 2*50=100 < 200 → no horizontal overflow
        XCTAssertEqual(out.height, 40)                        // 2*100=200 → maxY=(200-100)/2=50 → 40 passes
    }
```

```swift
import CoreGraphics

/// Non-destructive zoom/pan for the live display. `scale` 1.0 = fit whole frame,
/// clamped to [1, maxScale]; `offset` is the pan (points) clamped so the scaled
/// content always covers the view (no empty margins). Pure value type — the
/// SwiftUI view owns the gestures; this owns the clamp math.
public struct ZoomPanState: Equatable {
    public var scale: CGFloat
    public var offset: CGSize

    public static let maxScale: CGFloat = 8
    public static let fit = ZoomPanState(scale: 1, offset: .zero)

    public init(scale: CGFloat = 1, offset: CGSize = .zero) {
        self.scale = scale
        self.offset = offset
    }

    /// Clamp a scale to [1, maxScale].
    public static func clampScale(_ s: CGFloat) -> CGFloat {
        min(max(s, 1), maxScale)
    }

    /// Clamp `proposed` so the content (of `fittedContentSize` at scale 1, the
    /// letterboxed size under scaledToFit) still covers `viewSize` at `scale`.
    /// Per axis: overflow = scaled − view; maxOffset = overflow/2; clamp to
    /// ±maxOffset (0 when the axis doesn't overflow). Empty sizes → .zero.
    public static func clampedOffset(_ proposed: CGSize, scale: CGFloat,
                                     viewSize: CGSize, fittedContentSize: CGSize) -> CGSize {
        guard viewSize.width > 0, viewSize.height > 0,
              fittedContentSize.width > 0, fittedContentSize.height > 0 else { return .zero }
        let maxX = max(0, (fittedContentSize.width * scale - viewSize.width) / 2)
        let maxY = max(0, (fittedContentSize.height * scale - viewSize.height) / 2)
        return CGSize(width: min(max(proposed.width, -maxX), maxX),
                      height: min(max(proposed.height, -maxY), maxY))
    }
}
```

```swift
        GeometryReader { geo in
            @Bindable var model = model
            let uiScale = geo.size.height / BroadcastLayout.referenceHeight
            let fitted = fittedContentSize(imageSize: model.latestImage.map {
                                               CGSize(width: $0.width, height: $0.height) },
                                           in: geo.size)

                if let cg = model.latestImage {

                        .interpolation(.high)
                        .scaledToFit()
                        .scaleEffect(model.zoomPan.scale, anchor: .center)
                        .offset(model.zoomPan.offset)
                        .gesture(dragGesture(view: geo.size, fitted: fitted, model: model))
                        .gesture(magnifyGesture(view: geo.size, fitted: fitted, model: model))
                        .clipped()

                controlsOverlay(model: $model, view: geo.size, fitted: fitted)
            }
            .ignoresSafeArea()
            .background(configuresWindow ? AnyView(BroadcastWindowConfigurator()) : AnyView(EmptyView()))
```

Add helpers to the view:
```swift
    // Letterboxed size the image occupies at scale 1 under scaledToFit.
    private func fittedContentSize(imageSize: CGSize?, in view: CGSize) -> CGSize {
        guard let img = imageSize, img.width > 0, img.height > 0,
              view.width > 0, view.height > 0 else { return view }
        let s = min(view.width / img.width, view.height / img.height)
        return CGSize(width: img.width * s, height: img.height * s)
    }

    private func dragGesture(view: CGSize, fitted: CGSize, model: AppModel) -> some Gesture {
        DragGesture()
            .onChanged { g in
                let proposed = CGSize(width: model.zoomPan.offset.width + g.translation.width,
                                      height: model.zoomPan.offset.height + g.translation.height)
                // NOTE: accumulate from a drag-start snapshot to avoid compounding;
                // store the start offset in @GestureState or @State and add translation to it.
                model.zoomPan.offset = ZoomPanState.clampedOffset(proposed, scale: model.zoomPan.scale,
                                                                  viewSize: view, fittedContentSize: fitted)
            }
    }

    private func magnifyGesture(view: CGSize, fitted: CGSize, model: AppModel) -> some Gesture {
        MagnificationGesture()
            .onChanged { value in
                model.zoomPan.scale = ZoomPanState.clampScale(model.zoomPan.scale * value)
                model.zoomPan.offset = ZoomPanState.clampedOffset(model.zoomPan.offset, scale: model.zoomPan.scale,
                                                                  viewSize: view, fittedContentSize: fitted)
            }
    }
```
Implementer notes (get these right during manual validation):
- Use a `@GestureState`/`@State` drag-start snapshot so the pan accumulates from
  where the drag began, not compounding every `onChanged`. Same for magnify
  (snapshot the starting scale). MagnificationGesture's `value` is relative to
  the gesture start, so multiply the *start* scale, not the live one.
- **Scroll-wheel zoom** (Paul wants it): SwiftUI has no scroll-wheel gesture on
  macOS — add a small `NSViewRepresentable` that installs an
  `NSEvent.addLocalMonitorForEvents(matching: .scrollWheel)` (or overrides
  `scrollWheel(with:)`) over the image area and adjusts `model.zoomPan.scale`
  by the wheel delta, then re-clamps the offset. Keep it scoped to this view.
- Re-clamp the offset after every scale change (both gestures + slider + 100%).

- [ ] **Step 3: Add the controls overlay (slider + Fit + 100%) with auto-hide**

Add a `controlsOverlay` to the view. `Fit` sets `.fit`; `100%` sets scale so one
image pixel maps to one point (`imagePixelWidth / fittedContentWidth`, clamped);
the slider binds scale (1…8) and re-clamps offset:
```swift
    @ViewBuilder
    private func controlsOverlay(model: Bindable<AppModel>, view: CGSize, fitted: CGSize) -> some View {
        VStack {
            Spacer()
            HStack(spacing: 12) {
                Button("Fit") { model.wrappedValue.zoomPan = .fit }
                Slider(value: Binding(
                    get: { model.wrappedValue.zoomPan.scale },
                    set: { s in
                        model.wrappedValue.zoomPan.scale = ZoomPanState.clampScale(s)
                        model.wrappedValue.zoomPan.offset = ZoomPanState.clampedOffset(
                            model.wrappedValue.zoomPan.offset, scale: model.wrappedValue.zoomPan.scale,
                            viewSize: view, fittedContentSize: fitted)
                    }), in: 1...ZoomPanState.maxScale)
                    .frame(width: 180)
                Button("100%") {
                    if let img = model.wrappedValue.latestImage, fitted.width > 0 {
                        model.wrappedValue.zoomPan.scale = ZoomPanState.clampScale(CGFloat(img.width) / fitted.width)
                        model.wrappedValue.zoomPan.offset = ZoomPanState.clampedOffset(
                            model.wrappedValue.zoomPan.offset, scale: model.wrappedValue.zoomPan.scale,
                            viewSize: view, fittedContentSize: fitted)
                    }
                }
            }
            .padding(8).background(.black.opacity(0.55), in: .capsule).padding(.bottom, 24)
            .opacity(controlsVisible ? 1 : 0)
            .animation(.easeInOut(duration: 0.3), value: controlsVisible)
        }
        .onContinuousHover { _ in showControlsThenScheduleHide() }
    }
```
Add the auto-hide state + logic to the view:
```swift
    @State private var controlsVisible = true
    @State private var hideTask: Task<Void, Never>?

    private func showControlsThenScheduleHide() {
        controlsVisible = true
        guard configuresWindow else { return }   // embedded view: never auto-hide
        hideTask?.cancel()
        hideTask = Task { @MainActor in
            try? await Task.sleep(nanoseconds: 2_500_000_000)
            if !Task.isCancelled { controlsVisible = false }
        }
    }
```
On the embedded view (`configuresWindow == false`) `controlsVisible` stays true
(the guard returns before scheduling a hide); on the detached OBS window it fades
after 2.5 s of no pointer movement and returns on hover — so the stream stays
clean but remains framable via gestures (which work regardless of chrome
visibility). Initialize the detached window hidden-after-first-timer by calling
`showControlsThenScheduleHide()` in `.onAppear`.

## Notes for the implementer

- Keep the text `overlay(scale:)` OUTSIDE the `scaleEffect/offset` transform — only the astro image zooms, not the title/caption labels.
- Every scale change (gesture, slider, 100%) must be followed by an offset re-clamp via `ZoomPanState.clampedOffset(...)` so a zoom-out never leaves the image panned into empty space.
- `.clipped()` on the transformed image so a zoomed image doesn't paint outside the view bounds (and over the black background).
- Do NOT touch the snapshot/replay/master paths — zoom/pan is display-only.
- Auto-hide is ONLY for the detached OBS window (`configuresWindow == true`); the embedded Live view keeps controls visible.
- `@Bindable var model = model` inside the `GeometryReader` body gives the Slider a binding into the `@Observable` AppModel.
